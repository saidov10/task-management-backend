## Decisions Made

- **Default storage driver = `local`** (disk) for dev/test; `s3` for prod (AWS S3 / MinIO). The
  local driver is fully functional via internal `/internal/storage/*` routes so the presigned-URL
  flow is exercisable end-to-end without standing up MinIO.
- **Local internal storage routes are unauthenticated** — the unguessable UUID in the storage key
  is the capability, mirroring how S3 presigned URLs work. Acceptable for the dev/test driver;
  production uses real object storage. Path-traversal is guarded in `LocalStorageDriver.resolvePath`.
- **Attachment row is created on POST (before bytes arrive)** rather than via a separate confirm
  step — keeps the MVP lightweight and matches the TZ model (no `status` field). A client that
  never uploads can leave a row with no object; a cleanup pass can be deferred to Phase 4.
- **S3 driver uses static `@aws-sdk` imports** (packages installed) rather than lazy/dynamic
  imports — simpler and type-safe; the unused code path in local mode is negligible.
