## Decisions Made

- **Cursor pagination rewritten to Prisma-native** (`cursor`/`skip:1`) instead of the old `id > cursor` `where` clause: the old form only paged correctly under default `created_at` order. Native cursoring + an `id` tiebreaker in `orderBy` keeps paging deterministic for every `sort_by` field.
- **Priority sorts logically without extra mapping** because the `IssuePriority` Postgres enum is declared `none→urgent`; `ORDER BY priority` follows enum declaration order.
- **Grouped responses are not cursor-paginated** — board/column views load wholesale. Capped at 500 rows (`GROUP_FETCH_CAP`) as a safety bound. For `group_by=assignee`, an issue appears in every assignee bucket; unassigned issues use `key: null`.
- **Response schema serializes both shapes** via one object schema containing all of `data`/`next_cursor`/`group_by`/`groups` (fast-json-stringify omits absent props) rather than a fragile `oneOf`.
- **Recreated `.env`** (gitignored, absent this session) from `.env.example` with fresh `openssl`-generated JWT secrets so migrations/tests could run against the local (non-docker) Postgres on `:5432` with `postgres:12345`.
